Section 02
产品动态
AI 消费级产品
Claude Anthropic · Managed Agents 2026.05.14–20
  • MCP Tunnels（Managed Agents 新功能）：Agent 可通过私有网络隐秘访问内部 MCP 服务器，无需暴露到公网——企业级 Agent 合规落地的关键能力。
  • 两项隐私与安全增强：Managed Agents 新增数据隔离与审计追踪，进一步满足金融/医疗等高合规场景需求。
  • Opus 4.7 Fast Mode（研究预览）：通过 speed: "fast" 参数启用，在保持 Opus 质量的同时大幅降低延迟。
  • Claude for Small Business 发布：接入 QuickBooks / PayPal / HubSpot / Canva / DocuSign，现成工作流即开即用。
MCP Tunnels 的「私有网络穿透」对产品 UX 意味着：AI 功能可以在不暴露内部架构的情况下连接企业数据。用户感知是「AI 更懂我们的系统」，而不需要经历复杂的数据授权流程——降低的不只是技术门槛，更是用户的信任摩擦。
来源：9to5Mac
ChatGPT OpenAI · Codex Mobile + Finance 2026.05.14–20
  • Codex Mobile（iOS/Android，覆盖所有计划包括 Free）：在手机上管理代码任务——审批操作、查看 Diff、截图、测试结果，Codex 在 Mac 上跑，手机上实时监控。
  • Personal Finance Dashboard（Pro，美国）：安全连接银行/投资账户，统一看板展示支出/账单/净值/订阅，支持直接用自然语言提问。
  • File Library 扩展至 Free / Go 用户，新增跨计划存储管理与删除控制。
  • Apps Directory 正式移除 Beta 标签，AI 应用生态从实验转向常规。
Personal Finance 的 UX 设计给高敏感数据场景提供了范式参考：①先展示「连接了哪些账户」建立可见性；②统一看板让数据先变得可理解；③再激活自然语言问答能力。这个「可见性 → 理解 → 对话」三步走，是 AI 进入敏感场景的标准信任建立路径。
来源：help.openai.com
Gemini Google · I/O 2026 2026.05.19–20
  • Gemini 3.5 Flash 发布：超越 3.1 Pro 的 coding/agentic/multimodal 能力，输出速度 4× 快于同级前沿模型；Gemini Omni 同期发布，支持任意输入→任意输出（含视频生成）。
  • Neural Expressive 设计语言：Gemini 专属视觉体系——流体动效、触觉反馈、高饱和色彩、全新字体。与 Material 3 Expressive 并行，标志 Google 为 AI 应用建立独立于 OS 的设计语言。
  • Gemini App 全面重设计：药丸形输入框 + 单一「+」菜单；导航抽屉升级为全屏界面；Gemini Live 从全屏模式改为内联体验，无需切换上下文。
  • Gemini Spark 个人 Agent：主动代用户行动（预订、购物、信息追踪），Gemini 正式从「助手」转型为「代理人」。
  • AI Search 生成式 UI：根据用户意图动态调整结果展示形式，支持内联图片、叙述视频、时间轴与交互可视化。
Neural Expressive 是 Google 对「AI 原生应用该长什么样」的官方回答——与 Material Design 不同，它是专门为 AI 交互设计的语言：流动而非静态、情感化而非中性。Gemini Live 从全屏到内联的转变更值得关注：AI 对话从「占据整个屏幕」走向「嵌入当前上下文」，是 AI 交互从独占模式到协作模式的重要信号。
来源：9to5google.com
Kimi Moonshot AI · WebBridge 2026.05.15
  • Kimi WebBridge 发布：浏览器扩展让 AI Agent 直接操控网页——点击、输入、跨页面跳转，使用用户当前的登录会话，无需重新授权。
  • Kimi Code 与 Claude Code 均可通过 WebBridge 执行复杂的多步 Web 工作流：跨平台数据采集、量化金融回测、重复性网页操作自动化。
WebBridge 代表的「Agent 借用用户身份操作界面」模式，是 AI 与 Web 交互的新范式——不需要 API，直接复用用户已有的账户权限。这对产品设计提出了新命题：当 AI 可以代替人操作你的产品界面时，面向人设计的 UI 是否还够用？哪些流程需要为 Agent 专门优化（更少步骤、更机器友好的交互路径）？
来源：cntechpost.com
Section 02 · B
企业级 AI 产品 & 云平台
企业级 AI
Microsoft 365 Copilot Researcher Critique + Council · Multi-model 2026.05.14–20
  • GPT-5.5 正式进入 M365 Copilot，Claude 模型同步在 Word / Excel / PowerPoint 默认开启——微软首次将多模型并存作为生产力工具的默认体验。
  • Researcher Critique + Council：Critique 让 GPT 评审 Claude 的回答（反之亦然）；Council 并排展示两个模型的不同结论，由用户决策采纳哪个。Multi-model 对比 UI 首次成为企业产品标配。
  • SharePoint Copilot 驱动页面创建：描述需求即出完整草稿，AI 从「辅助编辑」升级为「起稿主力」。
Critique + Council 的「双模型对比界面」是今年最值得关注的 AI UX 创新之一——它打破了「AI 只给一个答案」的范式，让用户看见不同 AI 的不同理解，并主动参与决策。设计挑战在于：如何呈现差异（高亮冲突点 vs 并排全文）、如何帮用户快速判断（差异摘要）、以及何时该展示单一答案而非多个选项。
来源：thewincentral.com
Google Workspace May 15 Weekly Rollout · I/O 2026 2026.05.15–20
  • NotebookLM 接入自动化工作流：新增「Ask NotebookLM」自动化步骤，在 Workspace 自动化流程中直接调用 NotebookLM 的知识库检索能力。
  • Google Vids + AI 视频头像：将 Slides 一键转换为 AI 配音视频；可添加 AI 生成的发言人头像，脚本和语音均由 Gemini 生成，无需真人出镜。
  • Gmail Live + Docs Live（I/O 2026 宣布）：Gmail 支持语音提问收件箱；Docs 支持语音驱动起草——生产力工具输入方式向语音延伸。
Google Vids 的「Slides → AI 配音视频」是个值得关注的 UX 跳跃：文档不再只是文档，AI 可以把静态内容自动转化为更高传播效率的媒介形式。对我们设计报告、方案、文档类产品时的启示：输出格式的转换能力本身就是用户价值——帮用户「把东西做出来」之后，还可以帮他们「把它更好地传出去」。
来源：workspace.google.com · winbuzzer.com
云平台 · 设计视角
AWS Bedrock AgentCore Amazon · Managed Payments 2026.05.14–20
  • Managed Payments：Agent 可自主支付 API / MCP 服务器 / 其他 Agent 的费用，与 Coinbase 和 Stripe 合作，设置会话级消费上限。
  • WorkSpaces for AI Agents：Agent 可通过托管桌面安全操作 GUI 应用，实现对界面的自主控制。
AI「行动能力」首次延伸到花钱和操作桌面，UI 需要建立 3 层结构：① 授权确认（给 Agent 多大权限）→ ② 执行监控（Agent 在做什么）→ ③ 行为回溯（做了什么、花了多少）。缺少任意一层，用户感知就会失控。
来源：aws.amazon.com
Azure Intelligent Purview Microsoft · Copilot Studio Wave 1 2026.05.14–20
  • Intelligent Purview：实时对 AI Prompt 和响应执行 DLP 检测，合规管控下沉至每次 AI 对话层。
  • Confidential AI：机密计算满足金融/医疗等高安全场景推理需求，隐私保护从基础设施层延伸至 AI 推理层。
合规功能「隐性化」是云 AI 产品的核心 UX 挑战——用户需要感知到受保护，但不能被频繁的合规提示打断工作流。两种方案：① 状态常驻（角落徽章「合规保护中」，建立长期信任）② 仅在触发时出现（内容被拦截时给出轻量解释）。选择哪种取决于用户的安全意识与产品场景。

Figma Make Skills · May Release 2026.05.14–20
  • Make Skills 上线：用 Markdown 文件定义团队 AI 工作规范（命名规则、组件协议、交付流程），通过 /指令 在任意 Prompt 中调用——AI 使用从临时对话升级为可积累的知识体系。

  • 多屏生成：单次 Prompt 最多生成 5 个关联屏幕，保持页面间的导航逻辑和视觉一致性——AI 原生无限画布。

Canva Offline Mode · Claude Integration 2026.05.14–20
  • Claude 直接集成：在 Canva 内对话 Claude，不用切换到独立 Claude 窗口——「AI 在你的工具里」模式落地。

  • 专业设计套件扩展：新增 Affinity / Cavalry / Capture One / DaVinci Resolve 连接器，Canva 正在成为创意工作流的中心枢纽。
Canva 内置 Claude 是「AI 嵌入现有工具」vs「独立 AI 工具」博弈的一个信号：用户不需要切换上下文，AI 能力以插件形式出现在他们已经信任的界面里。这降低了 AI 的使用门槛，但也对 AI 功能的界面设计提出更高要求——它必须融入而非干扰当前工作流。

Section 04
对我们的启示
启示 01
Canva Claude 集成 + ChatGPT Personal Finance Dashboard
AI 进入敏感场景，必须先建立可见性
ChatGPT Personal Finance 的三步路径（可见性 → 理解 → 对话）揭示了 AI 进入高敏感数据场景的通用范式：先让用户看清楚「AI 连接了什么」，再帮他们理解数据，最后才激活问答能力。Canva 内置 Claude 也遵循同样逻辑——AI 嵌入已信任的界面，比独立入口更容易被接受。我们在引入 AI 功能时，应优先设计「数据可见层」，而不是直接跳到能力展示。
启示 02
Figma Make Skills · ChatGPT Codex Mobile
AI 使用正从「临时对话」升级为「可积累的工作流」
Figma Make Skills 用 Markdown 把团队 AI 工作规范固化为可调用的约定体系；ChatGPT Codex Mobile 把「手机委派 → Mac 执行 → 手机收结果」打通为异步工作闭环。两者都指向同一个趋势：AI 的价值正在从单次对话转向可沉淀的工作流系统。对我们的启示是：AI 功能的长期价值，不在于每次对话有多智能，而在于用户能否把经验积累下来，形成可复用的约定。
启示 03
Google I/O 2026 Gemini App 重设计 · pill 输入框 + inline Live
AI 入口应该是「常驻控件」，而不是「独立页面」
Gemini App 把输入框从全宽文本区压缩成悬浮 pill，并将 Live 实时视频直接嵌入对话流。这背后的设计信号是：AI 输入不再是「开启一段对话」的仪式，而是随时可触发的环境控件。对我们的启示：在云产品中引入 AI 功能时，不要新建独立的「AI 页面」——把它做成可从任意上下文快速唤起的浮层或快捷入口，才符合 AI 原生使用习惯，减少上下文切换成本。
启示 04
Google I/O 2026 Gemini Spark 个人 Agent
主动型 Agent 需要同时设计「打断预算」
Gemini Spark 不等用户提问，而是主动感知生活上下文、在合适时机推送建议——AI 交互从「人找 AI」翻转为「AI 找人」。随之而来的设计挑战是：什么时候介入才是帮忙而非打扰？通知应携带多少上下文才够可信？我们在设计主动型 AI 功能时，需要和能力本身一起设计「打断预算」机制——让用户能感知、控制 AI 的主动行为频率与边界，否则「智能」很快会变成「噪声」。
启示 05
Microsoft 365 Copilot Researcher Critique + Council 多模型对比
「呈现分歧」比「给出定论」更能建立用户信任
Microsoft 用 Critique 让模型自我质疑，用 Council 把 GPT 与 Claude 对同一问题并排输出让用户当评委。这是一个重要信号：当 AI 能力趋同，差异化转移到「如何呈现不确定性」。对我们的启示：在 AI 辅助决策场景中，与其输出一个自信的答案，不如把置信度、多角度、反驳意见一并呈现——尤其在云运维、费用分析等高风险决策场景，「AI 给出分歧供人判断」比「AI 给出定论」更能赢得信任。
启示 06
Anthropic MCP Tunnels + 云平台 Agent 权限管理演进
AI 的「连接管道」本身正在成为需要被界面化的产品体验
Anthropic 推出 MCP Tunnels，让 Claude 通过标准化协议安全接入第三方服务；各大云厂商同步强化 Agent 行为的审计与权限管理界面。这表明 AI 的「连接能力」不再只是技术后台，而是需要被产品界面化的核心体验。对我们的启示：「它能连什么、已连了什么、用户如何授权和随时撤销」这条链路的透明度，将直接影响用户是否愿意给 AI 更多权限。在云产品中，把 AI 连接状态做成可见、可控的一等公民 UI，是建立长期信任的基础。
