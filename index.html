AI 消费级产品

  • Claude Code 桌面版界面改版：会话侧栏 + 拖放工作区 + 集成终端，是目前最完整的 AI 原生 IDE UX 探索，侧栏式任务历史管理值得参考。

来源：9to5Mac
ChatGPT OpenAI · GPT-5.5 Instant 2026.05.05
  • Memory Sources 透明化面板：用户可查看 AI 调用了哪些历史对话/记忆/文件，并可逐条删除或纠正——把 AI 黑盒决策变成可检视的控制界面。
  • Fast Answers：AI 对高置信度问题直接给出深度答案，跳过推理链展示。置信度分级影响响应形式——这是答案质量信号化的 UX 尝试。
  • Trust Contact：用户可指定信任联系人，在需要支持时由 AI 主动触达——AI 主动行动场景的权限边界设计。
Memory Sources 的核心 UX 价值：不只是「记住你」，而是「让你看见 AI 记住了什么」。可见 + 可纠正，是比单纯个性化更高级的信任设计。
来源：openai.com · TechCrunch
Gemini Google · Android Show 2026 2026.05.12–13
  • Material 3 Expressive 设计语言：更强情感化动效、高对比色彩方案、流体过渡——Google 官方设计语言明确向「情绪化」转向，值得关注的移动端视觉趋势。
  • 自然语言自定义 Widget：用描述生成个性化桌面组件，UI 配置从「选项点击」升级为「说出来」——对话式界面配置的移动端验证。
  • Rambler 语音转结构化文字：随意语音口述 → 整理成书面段落，输入形态的 UX 革新。

Material 3 Expressive 的核心信号：Google 认为 AI 时代的 UI 需要更强的情感表达，不只是「功能正确」，还要「感觉对」。这对我们的设计语言演进方向有直接参考价值。
来源：blog.google · 9to5Google
Kimi · DeepSeek · MiniMax 中国 AI 产品 UI 观察 2026.04–05
  • Kimi：延续「内容优先」极简界面，文档/多文件管理 UX 是差异化——侧重让用户感知「AI 在读懂上下文」，文件引用可视化处理较好。
  • DeepSeek：推崇「裸模型直达」哲学——无 onboarding 引导、无推荐功能、无情感化包装，是截然不同的 AI 产品 UX 路线：把能力透明化，让用户自己建立使用模型。
  • MiniMax：聚焦角色化和情感化 UI，B 端场景定制化程度高；多模态输入（语音/图像）的交互集成比同类产品更流畅。
DeepSeek 的「零引导」设计是一个反直觉的成功案例——对有自驱力的用户，去掉所有辅助反而建立了更强的信任感。值得思考：我们产品的哪些引导是在帮用户，哪些只是在降低我们的焦虑？
来源：TechCrunch · DeepLearning.AI

企业级 AI 产品 & 云平台
企业级 AI
Microsoft Copilot Cowork + Agent Mode 2026.05.01
  • Copilot Cowork 移动端：iOS / Android 上随时委派任务，途中交代、回来收结果——AI 异步工作流的移动端完整闭环。
  • Copilot Agent Mode：在 Word / Excel / PowerPoint / Outlook / OneNote 内统一上线，支持迭代式协作创建文档，AI 成为每个工具的内嵌协作者。
「委派 → 离开 → 回来收结果」是移动端 AI 异步工作流的标准范式。关键 UX 问题：用户离开时如何维持掌控感？通知设计、进度回执、结果预览是三个值得打磨的触点。
来源：microsoft.com
Gemini for Workspace Google · Workspace Intelligence 2026.05
  • Docs 数据信息图生成：用企业私有数据一键生成可视化信息图，AI 直接产出设计成品而非辅助。
  • Slides 单 Prompt 生成完整演示：一句话生成完整可编辑 PPT，降低从零起稿的心理门槛（即将上线）。
  • Workspace Intelligence 自动学习用户写作风格，跨应用统一个人语气——风格记忆延伸至生产力工具。
Docs 信息图和 Slides 生成都在解决同一个问题：让「AI 直接出稿」而不是「AI 辅助你出稿」。这对设计工具的 AI 功能定位有参考：是让用户更快操作，还是直接替代某些操作步骤？
来源：workspace.google.com
ChatGPT Enterprise · Workspace Agents OpenAI 2026.05.05
  • 原生连接 Slack / Google Drive / M365 / Salesforce / Notion：企业数据直接驱动 Agent 工作流，无需中间层集成——AI 真正「进入」用户的工作环境，而不是另开一个窗口。
「AI 在你的工具里」vs「你去 AI 工具里」——两种截然不同的集成哲学。前者对用户的习惯侵入最小，是企业级 AI 产品更易接受的落地路径。
来源：callsphere.ai
云平台 · 设计视角
AWS Bedrock AgentCore Amazon · Managed Payments 2026.05
  • Managed Payments：Agent 可自主支付 API / MCP 服务器 / 其他 Agent 的费用，与 Coinbase 和 Stripe 合作，设置会话级消费上限。
  • WorkSpaces for AI Agents：Agent 可通过托管桌面安全操作 GUI 应用，实现对界面的自主控制。
AI「行动能力」首次延伸到花钱和操作桌面，UI 需要建立 3 层结构：① 授权确认（给 Agent 多大权限）→ ② 执行监控（Agent 在做什么）→ ③ 行为回溯（做了什么、花了多少）。缺少任意一层，用户感知就会失控。
来源：aws.amazon.com
Google Cloud · Agentic Data Cloud Cloud Next 2026 2026.05
  • Agentic Data Cloud：Agent 可跨云实时查询数据（含 AWS 上的数据），无需迁移，数据来源跨越多个平台。
  • Admin Console AI Control Center：管理员可精细管控 Agent 对 Workspace 各应用数据的访问权限边界。
当 AI 能跨云访问多个数据源，用户越来越难感知「这次回答用了哪些数据、从哪里来」。数据溯源 UI（来源标注 + 访问日志入口）将成为企业 AI 产品可信度设计的标配——不只是 IT 后台，而应该出现在用户侧。
来源：cloud.google.com
Azure Intelligent Purview Microsoft · Copilot Studio Wave 1 2026.05
  • Intelligent Purview：实时对 AI Prompt 和响应执行 DLP 检测，合规管控下沉至每次 AI 对话层。
  • Confidential AI：机密计算满足金融/医疗等高安全场景推理需求，隐私保护从基础设施层延伸至 AI 推理层。
合规功能「隐性化」是云 AI 产品的核心 UX 挑战——用户需要感知到受保护，但不能被频繁的合规提示打断工作流。两种方案：① 状态常驻（角落徽章「合规保护中」，建立长期信任）② 仅在触发时出现（内容被拦截时给出轻量解释）。选择哪种取决于用户的安全意识与产品场景。
来源：azure.microsoft.com

Section 04
对我们的启示
启示 01
Canva AI 2.0 记忆库 + ChatGPT Memory Sources
做 AI 个性化，必须同时做「可见性」
Canva 把记忆做成使用飞轮（越用越懂你），ChatGPT 把记忆做成透明面板（让你看见 AI 记住了什么）——两种策略相辅相成。我们在 AI 功能中引入行为沉淀时，应同步提供「AI 为什么这样做」的折叠入口。偏好累积 + 可纠正，是用户真正信任 AI 的前提，缺一不可。
启示 02
对话入口：Figma Make · Adobe Firefly · Canva
AI 对话正在成为每个工具的「第二界面」
Figma、Adobe、Canva 都在主工具之上叠加了一层自然语言入口。这不是替代原界面，而是给有明确目标的用户提供捷径。我们的 AI 功能入口设计面临同样的选择：是嵌入现有流程，还是独立一层？两者的用户门槛和场景覆盖截然不同。
启示 03
Claude Design + Figma AI 图像编辑
AI 生成功能应强制注入 Design Token，防止品牌漂移
Claude Design 读取 Figma 文件提取设计系统，Figma 内置 AI 工具均支持上传参考图锁定风格。我们在 AI 生成界面或内容时，应优先从产品已有的 Design Token / 样式规范取值——哪怕只是颜色和字体的强制对齐，也能大幅提升输出置信度，避免每次生成都要手动校正。
启示 04
Google Material 3 Expressive
AI 时代的设计语言需要更强的「情感表达」
Google 用 Material 3 Expressive 明确表态：功能正确已不够，UI 还要「感觉对」。情感化动效、高对比色彩、流体过渡——这些在理性主义设计浪潮中被压制的元素正在回归。当 AI 让功能实现越来越低成本，体验质感和情感共鸣将成为产品真正的差异点。
启示 05
DeepSeek 零引导哲学 + Figma Question Cards
引导设计是帮用户，还是在降低我们自己的焦虑？
DeepSeek 用「零引导 + 裸模型直达」建立了强信任；Figma 用 Question Cards 将模糊需求结构化——两种截然相反的策略都在各自场景中奏效。这提示我们：引导的价值取决于用户是否真的需要它。定期审查产品中的引导流程，区分「降低用户认知负担」和「掩盖产品复杂度」，是两件不同的事。
启示 06
AWS AgentCore + GCP Admin Console + Azure Purview
云平台正在把「AI 权限与行为」变成产品 UI 的核心命题
三大云厂商同时在做一件事：让 AI 的行动边界变得可管控——Agent 花多少钱、访问哪些数据、对话是否合规，全部下沉到平台能力层。这对我们的产品意味着：权限配置、数据溯源、操作回溯这三类 UI，将从「后台管理功能」升格为「用户侧可信度基础设施」。云基础能力越强，我们越需要在产品前台把这些能力的边界讲清楚——否则 Agent 能力越强，用户反而越不敢用。
