Section 01
本期速览
FIGMA
01
Figma AI Design Agent 发布——画布内嵌 Agent，点击任意元素即激活；元素级精准编辑（按钮、色值、图标），不破坏 Design Token；支持并行迭代多个风格方向，Agent 持续运行你同步编辑。设计协作进入「共编辑」时代。

来源：blog.google

Section 02
产品动态
AI 消费级产品
Claude Anthropic · Managed Agents Dreaming 2026.05.21–27
  • "Dreaming" 跨 Agent 持久化记忆：Agent 在任务间主动写结构化笔记；接班 Agent 读取笔记恢复上下文，规避已知错误——解决了 Agent「失忆」这一最大体验痛点。
  • 创意工具原生连接器：Claude 直接接入 Photoshop / Blender / Ableton / Splice / Autodesk Fusion，设计与创意工作流首次纳入 Claude 原生工具链。
  • Self-hosted Sandbox 公测：开发者可将工具执行路由到自有基础设施，Anthropic 负责编排——企业数据主权与 AI 能力兼得。
Dreaming 的核心 UX 价值不是「AI 更聪明」，而是「AI 不再重蹈覆辙」——用户不再需要在每次新会话里重新解释背景、重复纠正错误。对产品设计的启示：Agent 的「记忆可见性」将成为新的信任设计维度，用户需要能查看、编辑甚至删除 Agent 积累的笔记。
来源：technologyreview.com
ChatGPT OpenAI · Codex Goal Mode GA 2026.05.21–27
  • Codex Goal Mode 正式 GA：用户交付里程碑级目标，Codex 自主运行数小时至数天，支持人工检入点（check-in）和暂停控制。
  • Codex 登陆 ChatGPT 手机端：自主编码 Agent 覆盖移动场景，「委派 → 监控 → 收结果」的异步工作闭环从桌面扩展到手机。
  • ChatGPT Ads Manager 上线：自助式广告管理平台内置于 ChatGPT；Klarna 接入 1 亿商品的购物引擎——对话界面首次引入广告原生 UI 形态。
Goal Mode 引出了一个尚未被解决的 UX 命题：如何设计「多天自主任务」的检入界面？用户需要在不打断 Agent 工作流的情况下，快速判断进度是否符合预期、何时需要人工介入。进度摘要、置信度指示、暂停钩子，这些都是全新的设计对象。
来源：openai.com/codex
Gemini Google · Neural Expressive 全面上线 2026.05.21–27
  • Neural Expressive 全面推送：新设计语言正式向所有用户推出——浮动 pill 输入框、内联 Live 语音（不再需要全屏切换）、流体动效与触觉反馈、响应内容富媒体化（粗体摘要、内联图片、交互可视化）。
  • Gemini Spark Daily Brief：每日晨间 Agent 摘要——拉取日历、Gmail 和网页信息生成个性化简报，向 Ultra 订阅用户推送。
  • Personalization 实验功能：用户可选择接入 Google 搜索历史让 Gemini 个性化响应，在模型下拉菜单里以「开关」形式呈现。
Neural Expressive 从「全屏 Live 模式」改为「内联语音」，是 AI 交互从「模态独占」走向「情境嵌入」的信号。Personalization 开关放在模型选择器里，是一个把隐私控制与能力选择合并的 UX 决策——简洁但可能让用户混淆两者边界，值得持续观察。
来源：9to5google.com · androidauthority.com
Section 02 · B
企业级 AI 产品 & 云平台
企业级 AI
Microsoft Copilot Studio Computer-Using Agents GA · Workflow Canvas 2026.05.21–27
  • Computer-Using Agents GA：Agent 直接操控网页和桌面 UI，能感知并适应界面变化——不再因按钮位置或布局更新而失效。
  • 可视化 Workflow Canvas 重设计：统一画布替代原有多界面入口；「Agent 节点」可直接拖入工作流步骤，AI 逻辑与业务流程在同一张图上编排。
  • Real-time Voice Agents：可部署于 Dynamics 365 Contact Center 的实时语音 Agent，支持多语言、知识库集成，在 Copilot Studio 内直接配置和测试。
Computer-Using Agents 对设计系统提出了新要求：UI 不只要对人友好，还需要「机器可读」。当 Agent 通过 DOM 感知界面状态时，稳定的语义化组件结构（而不只是视觉一致性）成为设计系统的新价值维度。
来源：microsoft.com/copilot-blog
Microsoft 365 Copilot Claude Opus 4.7 + PDF in Chat 2026.05.21–27
  • Claude Opus 4.7 进入 M365 Copilot：用户可在 Word / Excel / PowerPoint 中选择 Claude Opus 4.7 处理复杂多步骤任务——多模型选择成为企业生产力工具标配。
  • PDF 在 Copilot Chat 内直接打开：无需跳转外部阅读器，上下文零切换，长文档问答体验大幅提升。
  • Outlook 邮件线程直接加入 Prompt 上下文：邮件可作为结构化上下文注入，Copilot 理解历史往来后直接生成回复草稿。
多模型选择器是今年最值得设计观察的 UI 元素之一——用户如何在不了解模型差异的情况下做出合理选择？是按任务类型分类（「复杂分析」「快速草稿」），还是展示性能/成本参数？目前微软的方案偏向直接命名，这个 UX 决策将在大量用户数据中得到验证。
来源：techcommunity.microsoft.com
Google Workspace Gmail Live · Docs Live · Style Matching 2026.05.21–27
  • Gmail Live：语音直接查询收件箱——问「我的航班登机口是哪里」，AI 从邮件中合成答案语音播报。
  • Docs Live：语音驱动写作——口述想法，Gemini 实时结构化、润色措辞，并从 Drive 和 Gmail 拉取上下文填充草稿。
  • Style Matching：Gemini 分析用户在 Drive 和 Gmail 的历史写作，跨应用统一语气与用词风格——AI 写作与个人声音趋同。
Style Matching 把「个性化写作」推向了一个新的信任边界：当 AI 能精准模仿你的写作风格，「这封邮件是你写的吗」将变成一个真实的沟通问题。产品设计需要建立新的「AI 代笔可见性」机制——不只是法律免责，而是帮助用户和协作者共同理解人与 AI 的创作边界。
来源：workspace.google.com
Section 03
设计工具动态
设计工具
Figma AI Design Agent · Beta 2026.05.21–27
  • 画布内嵌 AI Design Agent 发布：点击任意画布元素激活 Agent（星形图标指示），自然语言驱动单个元素或全局批量修改，不覆盖 Design Token 和组件定义。
  • 并行迭代模式：一次 Prompt 可请求多个风格方向（如「有机感」「现代感」「复古感」），Agent 并发迭代，用户同步继续编辑——人与 AI 真正进入「共编辑」状态。
  • 双模型驱动：同时集成 OpenAI 和 Anthropic 模型；Beta 期间免费使用 AI Credits，正式版进入 Professional / Org / Enterprise 计划。
这是本期最重要的设计工具变化。Figma Agent 与此前所有「AI 设计工具」的本质区别在于粒度：不是「生成一个屏幕」，而是「修改这个按钮的圆角值」。元素级精准编辑 + 尊重设计系统，意味着 AI 第一次可以安全地进入生产级设计文件，而不只是原型探索阶段。
来源：figma.com/blog · fastcompany.com
Canva AI 2.0 Design Model · Code 2.0 2026.05.21–27
  • Canva Design Model 上线：全球首个专为「设计结构、层级与复杂度」训练的基础模型，生成完整可编辑分层稿件，自动应用品牌 Token——输出即是可交付的设计文件，而非图片。
  • Brand Intelligence：自动从连接的品牌模板中提取字体、颜色和风格，确保每次生成都符合品牌规范，无需人工逐项校正。
  • Canva Code 2.0：对话式生成响应式交互页面，新增 HTML 导入编辑功能，Canva 正在成为轻代码可视化内容的完整生产链路。
Canva Design Model 的关键 UX 突破是「生成即可编辑分层稿」——打破了 AI 生成工具长期存在的「好看但不可用」困境。当输出已经是带 Token 的分层文件时，设计师的工作从「重建 AI 的输出」变成「精修 AI 的草稿」，工作流位置发生了根本性变化。
来源：canva.com/newsroom
Google Search 25 年搜索框大改版 · 多模态输入 2026.05.21–27
  • 搜索框彻底重设计：Google 替换了沿用 25 年的单行文字输入框，新输入框同时接受文字、图片、文件、视频、Chrome 标签页——多模态输入成为默认交互范式。
  • 信息 Agent（Information Agents）：用户可创建 AI Agent 持续监控特定主题的网页动态，主动推送关键变化——搜索从「主动查询」变成「被动订阅」。
  • AI Mode 已覆盖 10 亿月活用户，Gemini 3.5 Flash 作为默认模型在全球最大的搜索界面运行。
这是 UX 设计史上影响规模最大的输入框改版。多模态输入进入 1B+ 用户的默认交互后，设计系统对「输入」的定义需要扩展——文件拖入、截图粘贴、标签页分享，都将成为用户在任意搜索式界面期待支持的操作。我们产品的搜索和提问入口，值得以此为参照重新审视。
来源：blog.google
Section 04
对我们的启示
启示 01
Figma AI Design Agent
AI 设计协作的价值在「元素级共编辑」，不在「屏幕级生成」
Figma Agent 打破了过去所有 AI 设计工具的默认模式——不是生成一整张屏幕，而是精准修改「这个按钮的圆角」「这组文字的色值」。元素级粒度 + 尊重 Design Token + 人 Agent 同步编辑，让 AI 第一次可以安全进入生产级文件。对我们的启示：在产品中引入 AI 生成能力时，颗粒度越细、越尊重现有系统约束，用户就越愿意在真实工作流中采纳它——而不只是用它做「一次性探索」。
启示 02
Google 搜索框改版 · 多模态输入成为主流
多模态输入正在从「特色功能」升级为「基础预期」
Google 把沿用 25 年的搜索框换掉，让 1B+ 用户的默认输入同时支持文字、图片、文件、视频和浏览器标签页。这个信号意味着：多模态输入不再是高端 AI 产品的特色，而是用户在任意搜索和问答界面开始期待的基础能力。对我们的产品：无论是运维查询、费用分析还是报告生成入口，如果只支持文字输入，将很快显得落后——截图粘贴、日志文件拖入、选中上下文发送，这些都应该进入设计规划。
启示 03
OpenAI Codex Goal Mode · Microsoft Computer-Using Agents
长时 Agent 需要「检查站 UX」——一个还没有人解决好的设计难题
Codex Goal Mode 运行数天，Copilot Studio Computer-Using Agents 持续操控界面——这类长时 Agent 引出了全新的 UX 命题：用户如何在不打断 Agent 的前提下，知道它在做什么、做对了没有、什么时候该人工介入？现有产品的答案（进度条 + 日志流）对大多数用户来说太低层。「检查站」UX 的设计挑战在于：在正确时机以正确粒度呈现 Agent 状态，让人既能放心等待，又能在出错时及时叫停。
启示 04
Claude Managed Agents "Dreaming" 跨 Agent 持久记忆
Agent 记忆需要「可见、可编辑、可撤销」——这是下一个信任设计维度
Claude 的 Dreaming 功能让 Agent 自主积累跨任务笔记，解决了 Agent 失忆的体验问题。但随之而来的设计挑战是：用户需要知道 Agent 记住了什么，并且能够干预。「AI 记住了我说过不喜欢某种方案」——如果用户无法查看和修改这些记忆，隐性偏好会悄悄影响所有后续输出。对我们的启示：任何形式的 AI 个性化记忆，都需要配套「记忆管理 UI」，让用户看见、确认和清除——否则个性化会演变成不透明的黑盒。
启示 05
Microsoft Copilot Studio Computer-Using Agents · Google Gemini Spark
Computer-Using Agents 让「机器可读的 UI」成为设计系统新要求
Copilot Studio 的 Computer-Using Agent 会直接操控网页和桌面 UI，并能感知界面变化而自我适应。这意味着设计系统的稳定性不再只是视觉一致性，而是语义结构的可靠性——组件的交互角色（这是「提交按钮」还是「取消按钮」）需要通过语义化的属性来表达，而不只是依靠视觉位置。对云产品设计的启示：随着 Agent 进入日常 SaaS 产品，可访问性（Accessibility）属性、ARIA 标注和语义化 HTML 结构，将从「合规要求」升格为「AI 可用性基础设施」。
启示 06
Google Workspace Style Matching · Perplexity Health · Salesforce Agentforce Coworker
AI「拟人化」正在触及三条新边界：声音、健康、同事身份
本周出现了三个 AI 拟人化的新极端：Google Style Matching 让 AI 模仿你的写作声音；Perplexity Health 接入你的医疗记录；Salesforce Agentforce Coworker 让 AI 出现在你的团队协作工具里以「同事」身份存在。这三个方向都需要全新的信任与同意设计框架：声音的归属（这份文档是谁写的？）、健康数据的权限（AI 可以看什么、说什么？）、同事关系的边界（Agent 可以被其他人看见吗？）——目前产品界还没有成熟的答案，但这是 2026 H2 UX 设计最值得关注的伦理命题。
