AI 消费级产品

Claude · ChatGPT · Gemini · Kimi · DeepSeek · MiniMax — UX 变化与功能发布

Claude Anthropic 2026.06.23
  • Claude Tag：常驻 Slack 的 AI 同事 —— 每个 Slack 频道都可以有一个 Claude 身份；用户可 @Claude 分配任务，也可让它以 ambient 模式持续跟进频道对话，主动积累团队上下文与机构知识。
  • Ambient 模式 vs. Task 模式 —— ambient 模式下 Claude 静默跟进、不打扰，随时可被唤醒；task 模式下 Claude 将任务拆解成阶段，在 Slack 线程中透明展示执行进度，所有人可见、可介入。
  • 组织知识积累 —— 同一 Claude 身份跨成员共享，上一位同事的对话结果下一位可直接接续；管理员配置 Claude 可访问的工具与数据范围，合规与透明双向保障。
Claude Tag 揭示了企业 Agent UX 的核心范式：AI 不再是一次性工具，而是有记忆的「同事」——设计任何团队 AI 功能时，需要明确 AI 是「静默感知/按需唤醒」还是「主动汇报/任务执行」，两种模式的交互入口、通知密度、数据可见性设计完全不同。
来源：techcrunch.com · Anthropic's Claude Tag is learning your company, one Slack message at a time

本期暂无重大 UX 更新。OpenAI 本周发布了开源安全倡议（使用 AI 检测修复开源漏洞），属于基础设施层面，暂无直接的产品交互变化。

Gemini Google 2026.06.23 · 06.18
  • Sheets 公式自动修复 —— Workspace 用户点击错误单元格 → 界面弹出「Fix」按钮 → Gemini 在侧边栏诊断原因并呈现修复方案 → 一键接受应用。把 AI 从「你去找 AI」变成「AI 在你需要的地方等你」。
  • Google Docs AI 强制曝光引发反弹 —— AI 写作建议以底栏 + 光标悬浮两种形式出现，关闭路径深埋在 Workspace 设置中，TechCrunch 专题指引引发大量转发，「如何关掉 AI」成为本周高搜词。
两个案例对照鲜明：Sheets 是「对的时机、对的地方」的 inline AI 设计典范；Docs 是「强制曝光 + 关闭路径不清晰」的负面教材。同一团队、同一周，形成了最直接的 UX 设计对比实验。

ChatGPT Enterprise · Claude Enterprise · Gemini for Workspace — UX 变化与团队协作

企业级 AI 平台
Claude Enterprise · Claude Tag Anthropic 2026.06.23
  • 企业 Slack 深度集成 —— 系统管理员配置 Claude 可访问的工具（代码库、销售数据、文档）和数据范围；Claude 以统一身份出现，任何团队成员均可查看 Claude 的工作历史并接续对话。
  • 透明执行 + 共享可见性 —— Claude 在 Slack 线程中拆解任务、逐步汇报，全频道可见执行状态；任何人可随时介入、接管或修改，避免「黑盒 Agent」的信任问题。
  • 组织知识沉淀 —— 随着 Claude 持续跟进频道，它积累的团队知识可被所有成员共享，形成可持续增长的「AI 知识库」，而非每次重新上下文。
企业 AI 产品的核心 UX 挑战：如何让 AI 的工作对团队「透明可信」？Claude Tag 的答案是「公开执行 + 共享历史 + 可随时介入」——这三点是企业级 Agent 与消费级 Agent UX 设计的本质区别。
来源：techcrunch.com · Claude Tag enterprise Slack integration
Gemini for Workspace Google 2026.06.23
  • Sheets inline AI 修复 —— 面向 Workspace 企业用户首先推出；将 AI 能力嵌入日常办公工具的错误处理流程，而非要求用户主动切换到 AI 入口。
  • 触发点设计的企业意义 —— 在企业场景下，AI 在「用户遇到问题的那一刻」出现，而非要求员工主动学习使用新工具，大幅降低采纳摩擦。
「AI 在哪里出现」比「AI 能做什么」更决定企业采纳率——把 AI 嵌入员工已有的错误处理路径，是企业产品提升 AI 功能渗透率最高效的设计选择。

设计主力工具

Figma · Adobe CC — AI 功能迭代与新交互范式

Figma Weave Figma · Runway 2026.06.18
  • Runway Aleph 2.0 集成 Figma Weave —— 设计画布内支持视频创作导演：带入参考图影响整体画面风格，最长 30 秒、关键帧编辑跨帧保持角色/场景一致性。
  • 「设计迭代」范式替代黑盒生成 —— 每次修改可预览、可撤销，逐步决策而非一次性生成；将视频创作从「黑盒输出」转变为「可控的渐进式创作流程」。
  • Figma 从静态设计工具延伸为多模态创意画布 —— 设计师不再需要离开 Figma 就能完成从静帧到动态视频的全链路创作，工具边界进一步扩展。
视频生成工具的关键 UX 进化：不是「生成更好的视频」，而是「让创作过程像设计一样可控」——参考图引导 + 逐帧决策 + 可撤销修改，是生成式工具从「实验性玩具」走向「专业工作流」的设计路径。
来源：figma.com/blog · Direct Every Frame with Runway Aleph 2.0
Adobe Firefly AI Studio Adobe 2026.06.18
  • Elements：可复用资产库 —— 已创建的角色、场景、对象可以命名保存为资产，在后续任意生成中直接引用，解决 AI 生成「每次重头、无法一致」的核心痛点。
  • Projects：跨会话工作流 —— 资产、生成物、创作上下文统一归档；关闭再打开从同一状态继续工作，结束了生成式 AI「每次无状态」的体验断裂。
  • AI 助手生成 logo + 视频草切 —— Firefly AI 助手扩展支持品牌 logo 生成和视频片段拼接成草稿，创作链条进一步延伸。
生成式 UI 产品的成熟标志：从「每次从零生成」进化为「带记忆的持续创作」——Elements（资产复用）+ Projects（会话持久化）是让 AI 创作工具从「实验性体验」变为「专业工作室」的两个核心 UX 设计决策。
来源：theverge.com · Adobe's redesigned AI studio remembers what your creations look like

设计师高频效率工具

Cursor · Notion AI · Canva AI — 值得关注的新功能与交互设计

Cursor · AI Agent Loop 效率工具生态 2026.06.22
  • Agent Loop 范式 —— Cursor CEO Arash Ferdowsi 公开描述三阶段演进：人写代码 → Agent 写代码 → Agent 驱动 Agent 写代码；多个 Agent 在后台持续循环（架构优化、代码去重、自动提 PR），无需人工干预。
  • 后台自主工作的 UX 挑战 —— 当 Agent 在人不在场时持续工作并直接提交 PR，「可中断性」「可审查性」「可回滚性」成为 Agent 产品的核心 UX 约束，而非可选功能。
  • 对设计工具生态的信号 —— Figma MCP Server（Agent 直接在画布上操作）和 Agent Loop 所描述的范式正在汇合——设计工具也将面临「Agent 在后台自主设计」的 UX 设计命题。
Agent Loop 让「监督界面」从加分项变成必选项：当 Agent 自主在后台循环工作，产品必须提供一个清晰的「当前在做什么、做了什么、可以撤销什么」的控制面板——否则用户会因失去掌控感而放弃使用。
来源：techcrunch.com · The AI world is getting 'loopy'
iOS 27 系统级 AI Apple 2026.06.21
  • 超越 Siri 的实用 AI 功能 —— System Orchestrator 可跨 App 全系统操作；AFM Framework 开放给开发者调用本地模型，AI 能力首次从系统级延伸至第三方 App。
  • 开发者 API 开放的设计含义 —— 设计师在 iOS 上构建 AI 功能时，可以调用系统级 AI 能力而非完全自建，但同时需要设计「系统 AI 与自建 AI」的能力边界与 UI 一致性。
平台级 AI API 开放对设计师的实际影响：评估哪些 AI 功能可以「委托」给系统层（System Orchestrator / AFM），哪些必须自建以保持产品差异化，是 iOS 27 时代 AI 功能选型的核心决策框架。

板块四
对我们的启示

从本期产品动态中提炼的 6 条具体、可执行的设计建议

Insight · 01
↗ Claude Tag ambient mode
设计 AI 的「环境感知」模式：静默跟进，按需唤醒
Claude Tag 的 ambient 模式揭示了一种新的 Agent UX 原语：AI 不等指令，而是持续感知上下文、在合适时机主动反馈。设计 AI 功能时，先定义它是「问才答」（被动模式）还是「持续观察」（环境感知模式）——两种模式的通知密度、权限设计、用户控制逻辑完全不同，不能混用。
Insight · 02
↗ Gemini Sheets 公式修复
操作触发型 AI：把 AI 放在用户「需要帮助的那一刻」
Gemini Sheets 的核心设计决策不是「做了什么 AI 功能」，而是「在哪里触发」——错误发生的那一刻弹出修复提案，把 AI 从侧边栏工具变成流程中的自然节点。设计 AI 功能时问：用户在哪个操作步骤最需要帮助？把 AI 入口放在那里，而不是放在「工具栏」。
Insight · 03
↗ Adobe Firefly Elements + Projects
生成式 UI 的持久状态：资产复用 + 会话持久化
Adobe Firefly 用 Elements（角色/场景资产库）+ Projects（跨会话上下文）解决了生成式工具最大的 UX 痛点：「每次从零开始，无法保持一致性」。任何生成式 AI 功能，在设计之初就需要规划：用户创建的内容如何保存？如何在下次使用时直接复用？没有持久状态的 AI 生成工具，用户黏性极低。
Insight · 04
↗ Google Docs AI 关闭指南
AI 功能的控制权设计：关闭路径与开启路径一样重要
Google Docs 的案例证明：当 AI 功能强制曝光而关闭路径不清晰，用户的反应不是「适应」而是「搜索如何关掉它」。设计 AI 功能时，「关闭/跳过」的入口应与功能本身同等可见；用户拒绝 AI 建议的操作应该简单到一个手势。控制感是信任的前提，不是可选项。
Insight · 05
↗ Figma Weave + Runway Aleph 2.0
生成式工具的「设计范式」：可撤销 + 逐步决策 + 参考约束
Runway Aleph 2.0 在 Figma 中的 UX 设计——参考图引导风格、每次修改可预览可撤销、逐帧决策——正是把生成式工具从「赌博式一次性输出」变为「像设计一样可控的迭代流程」的关键。任何生成式功能都应具备：可以约束生成方向（参考输入）、可以逐步验证（分阶段预览）、可以局部修改（不全盘重来）。
Insight · 06
↗ Cursor CEO · Agent Loop
Agent 自主工作时，监督界面是产品的核心功能
当 Agent 在后台自主循环工作（优化架构、提交 PR、修改设计），「监督界面」从附加功能变成核心功能：用户需要清楚地看到 Agent 当前在做什么、已经做了什么、有什么可以撤销。设计 Agent 类产品的最小标准：一个清晰的「Activity Log」+ 明显的「暂停/撤销」入口，缺一不可。
